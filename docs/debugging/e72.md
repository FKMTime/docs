# E72 — NVS_BUZZER_VOLUME_WRITE_FAILED

:::note Hardware version
This error only occurs on **v4** hardware.
:::

Writing the buzzer volume setting to NVS failed.

## What happens

The error is logged **once per boot**. The menu still closes and the new volume is applied in RAM for the current session. After a reboot, the volume will revert to the previously saved value (or the default if nothing was saved).

## Possible causes

- The NVS partition is full.
- A flash write error.

## How to fix

1. **Retry changing the volume** — After a reboot, try setting the volume again. A transient write error may not recur.
2. **Erase NVS and reflash** — If NVS writes consistently fail, perform a full flash erase and reflash.
