# E55 — BLE_REQUEST_SECURITY_FAILED

Requesting BLE security (encryption) on the connection failed.

## What happens

The current connection attempt is abandoned (`continue 'outer`) and the device retries the connection loop immediately. The error is recorded **once per boot**.

## Possible causes

- The BLE stack rejected the security request — this can happen if the connection dropped at the exact moment the request was made.
- The FKMTime Display rejected the security request.

## How to fix

1. **Wait** — The connection loop retries automatically.
2. **Clear the bond and re-pair** — If this error appears repeatedly, delete the stored bond and pair the FKMTime Display again from scratch.
3. **Contact support** — Open an issue on [GitHub](https://github.com/FKMTime/firmware).
